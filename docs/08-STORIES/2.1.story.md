# STORY-2.1: Admin — Visão de Membros Pendentes

## Contexto

Fabio (responsável pelo digital) tem 39 contatos que precisam ser convidados a completar o cadastro. Atualmente a página `/admin/pre-registrations` mostra pré-cadastros pelo processo, mas **não mostra o grupo do membro** e não permite ver quem ainda não fez o primeiro login de forma clara.

Fabio precisa ver: **nome, grupo, status de acesso** e poder **gerar mensagem WhatsApp** por membro com as credenciais de acesso.

## Problema atual

A página `/admin/pre-registrations` lista pré-cadastros (processo), mas:
- ❌ Não mostra qual grupo cada membro pertence
- ❌ Não tem filtro "nunca logou" de forma destacada
- ❌ Não tem visão consolidada dos 39 pendentes

## Acceptance Criteria

- [ ] A tabela de pré-cadastros exibe coluna **Grupo** (nome do grupo de consórcio do membro)
- [ ] Existe filtro/badge para mostrar apenas membros que **nunca fizeram login**
- [ ] Cada linha exibe: nome, telefone, grupo, status (Nunca acessou / Ativo / Expirado)
- [ ] Botão por linha: "Gerar mensagem WhatsApp" → abre modal com mensagem pronta para copiar
- [ ] A mensagem WhatsApp inclui: nome do membro, usuário, senha temporária, link do site
- [ ] Contador no topo: "X membros aguardando primeiro acesso"
- [ ] Busca funciona por nome, telefone OU nome do grupo

## Tarefas de Implementação

### 1. API — adicionar grupo ao endpoint

- [ ] `GET /api/admin/pre-registrations` — incluir `group_name` via JOIN com `quotas` + `groups`

### 2. Tabela — nova coluna Grupo

- [ ] `components/pre-registrations/PreRegistrationsTable.tsx` — adicionar coluna Grupo
- [ ] Status badge: "Nunca acessou" (cinza) | "Ativo" (verde) | "Expirado" (vermelho)

### 3. Filtro de pendentes

- [ ] Toggle/tab "Todos | Pendentes | Ativos" na página de pré-cadastros

### 4. Contador no header

- [ ] `PageHeader` action area: badge com total de pendentes

### 5. Busca estendida

- [ ] Filtrar também por `group_name` no `searchTerm`

## Arquivos a Modificar

| Arquivo | Tipo |
|---------|------|
| `app/api/admin/pre-registrations/route.ts` | JOIN com groups |
| `components/pre-registrations/PreRegistrationsTable.tsx` | Coluna grupo + status badge |
| `app/(protected)/admin/pre-registrations/page.tsx` | Filtros + contador |

## Teste Manual

1. Abrir `/admin/pre-registrations`
2. Verificar que coluna Grupo aparece para cada membro
3. Clicar em filtro "Pendentes" → ver apenas quem nunca logou
4. Clicar em "Gerar mensagem WhatsApp" → copiar mensagem com credenciais
5. Buscar por nome de grupo → filtrar corretamente
